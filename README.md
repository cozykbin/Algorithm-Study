## 트랙 운영 방식 (A형 / B형)

### 트랙 선택
- 본인 목표에 맞춰 A/B형 중 선택 

### 🟦 A형 트랙
- A형 대비 
- 유형: 구현, 시뮬레이션, 단일 BFS/DFS, 기본 자료구조  

### 🟥 B형 트랙
- B형 대비
- 유형: 그래프, DP, 백트래킹, 시뮬+조합, 최단거리, 고급 자료구조  

---

## 📅 주간 스터디 운영 흐름  

### 월요일 — **주간 문제 선정**
- **트랙별 주간 문제 6문제** 선정  
- 선정된 문제는 `Issues` 에 공지
---

### 월~토요일 — **하루 1문제 풀이 DAY**
- **A형/B형 중 본인 트랙 문제를 하루 1개 해결**
- 최소 1시간은 혼자 고민 (AI 금지)
- 못 풀어도 OK → 아래 기록만 남기면 충분  
  - 접근 방식  
  - 시도했던 풀이  
  - 막힌 지점  
- 이후 참고한 풀이/코드는 반드시 **내 말로 정리해서 PR 제출**

---

### 일요일 — **실전 연습 DAY (2시간 집중 풀이)**
> 오프라인에서 실전처럼 한 문제를 **2시간 동안 풀어보는 시간**

- 2시간 동안  
  - 코드 작성  
  - 최적화  
  - 디버깅  
  - 결과 제출  
- 끝나고 함께 간단 회고 진행  
  - 어떤 접근이 통했는지  
  - 어디서 막혔는지  
  - 다음 주 문제 선정에 반영

---

### 일요일 밤 ~ 월요일 새벽 — PR 마감 정리
- 주간 PR 정리 완료  
- 하이라이트 코드 / 회고 내용은 스터디 저장소에 아카이브  
- 다음 주 준비

---

## ✍️ 스터디 규칙

1. 백준 스트릭 끊기지 않기  
2. 문제당 1시간 이상 고민 → 이 시간은 **AI 금지**  
3. 고민 후 참고할 때는 반드시  
   - 내가 이해한 말로 정리  
   - PR에 기록으로 남기기  
4. 서로의 코드/사고방식을 리뷰하며 성장하기

### Commit 규칙
> `[문제플랫폼] 문제이름 / 난이도 / 걸린시간` 형식으로 작성

- **터미널에서 작성법:**
  ```bash
  git commit -m "[BOJ] 미로 탐색 / Silver1 / 45분"
  ```
- **플랫폼 작성법 통일:**
  - `[BOJ]` - 백준
  - `[PGS]` - 프로그래머스
  - `[LTC]` - 리트코드
  - `[SEA]` - 삼성SW Expert Academy
  - `[ETC]` - 그 외

<br>

### Pull Request(PR) 규칙
> PR 제목: `[아이디] O월 O일 문제풀이`

- **예시)**
  - `[cozykbin] 10월 2일 문제풀이`
- **Comment :**
  - 오늘 풀었던 문제에 대한 간단한 회고(어려웠던 점, 새로 알게 된 점 등)

<br>

### 코드 리뷰 규칙
- **(예시)**
- 이런 방식으로 접근하신 게 정말 좋은 것 같아요!
→ 잘했다고 생각하는 부분 / 인사이트

- 이 변수명을 조금 더 명확하게 바꾸면 어떨까요?
→ 가독성, 네이밍, 구조 개선 제안

- 이 부분은 어떤 의도로 작성하셨는지 궁금해요!
→ 이해가 안 가는 부분 질문

- 이 문제는 이런 다른 풀이법도 있더라고요!
→ 다른 접근 / 자료 공유
